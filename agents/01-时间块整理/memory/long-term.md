# 长期记忆

> 本文件记录 Agent 在长期运行中积累的经验、用户偏好和注意事项。

## 1. 用户偏好

### 1.1. 任务类型标准写法
- **focus**: 专注任务，需要计时的核心工作
- **break**: 休息时间，包括做饭、锻炼、午休等
- **other**: 其他类型任务

### 1.2. 常用项目名称和缩写
- **Life-OS / life-os**: 生命操作系统项目
- **ExoMind / exomind**: 外心项目（原名）
- **vibe-kanban**: 任务管理工具
- **lean-spec**: 需求管理工具

### 1.3. 反馈中常见关键词含义
- **身心感觉**: 用户的主观状态描述，包含情绪和身体感受
- **反馈总结**: 用户对任务完成情况的客观评价
- **2.1**: 用户评价系统的评分前缀，数字越高评价越好

### 1.4. 格式偏好
- 时间块之间使用 `---` 分隔
- YAML 元数据使用 ```yaml 代码块
- AI 反馈使用 `> [!ai]+` callout 块
- 任务标题格式：`# HH点MM分 任务名称`

### 1.5. AI 反馈可视化原则 ⭐📚🌡️📋🎯
| 原则 | 具体做法 |
|------|----------|
| 🏷️ 标题高亮 | 重要概念用 `==高亮==` |
| 🎨 可视化 | 标题加 emoji（⭐📚🌡️📋🎯） |
| 📏 精简 | 每条内容控制在 20 字以内 |
| 🔗 链接 | 资源用 `[链接文字](URL)` 格式 |
| 🎯 重点突出 | 关键信息用 `==` 标记 |
| ✂️ 精简原则 | 内容减少约 50%，只保留核心信息 |

**示例格式**：
```markdown
> [!ai]+ AI 反馈
> **任务概要**: 与 ==Claude/Opus== 讨论 ==Life-OS 架构设计==...
>
> **==完成事项==** ⭐
> - 🏗️ 目录结构：`core/human/agents/projects/workflows`
> - 🧠 核心：人≠Agent，人有自主性
```

## 2. 经验积累

### 2.1. 常见的语音识别错误模式
| 错误类型 | 示例 | 修正方法 |
|----------|------|----------|
| 同音字替换 | 技击桩 → 击击装 | 根据上下文判断 |
| 专有名词错误 | CodeX → odex | 保持原样或标注存疑 |
| 时间格式错误 | 17点07分 → 70点07分 | 识别 HH点MM分 模式 |

### 2.2. 某些任务类型的典型时长
- 短任务（思考/简单操作）: 5-15 分钟
- 中任务（设计/整理）: 30-60 分钟
- 长任务（编码/写作）: 1-2 小时

### 2.3. 用户反馈中的高频模式
- 「焦虑」「急躁」→ 可能是空想太多、行动太少
- 「舒服」「好」→ 任务进展顺利
- 「累」「困」→ 需要休息

## 3. 注意事项

### 3.1. 必须保留原始内容
- 用户的主观判断和情绪表达
- 专有名词的原始写法（如人名、项目名）
- 用户明确标注的「存疑」内容

### 3.2. 可修正的内容
- 明显的语音识别错误（如同音字）
- 语法不通顺但意图明确的句子
- 格式不规范但可识别的结构

### 3.3. 特殊处理
- Git 相关操作需要记录到 .logs/
- 待办事项仅列出，不写回其他文件
- 必须执行 git commit，不能跳过
