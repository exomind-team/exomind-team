## 协作模式

### 评审流程标准

1. AI 评审阶段 → 自动检查完整性
2. 人工评审阶段 → 等待团队成员审核
3. 合并 → 更新劳动日志

### Git 提交规范

- 分支格式：`feature/YYYY-MM-DD-任务标识`
- Commit 格式：`feat: [任务类型] [任务摘要]`
- 必须创建 PR，不直接提交主分支

### PR 模板要素

- 文档概述
- 变更内容
- 评审状态
- 关联劳动记录
- 检查清单
