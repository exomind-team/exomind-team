## 协作模式

### 评审流程标准

1. AI 评审阶段 → 自动检查完整性
2. 人工评审阶段 → 等待团队成员审核
3. 合并 → 更新劳动日志

### Git 提交规范

- 分支格式：`feature/YYYY-MM-DD-任务标识`
- Commit 格式：`feat: [任务类型] [任务摘要]`
- 必须创建 PR，不直接提交主分支

### PR 模板要素

- 文档概述
- 变更内容
- 评审状态
- 关联劳动记录
- 检查清单

## 经验教训（重要！）

### 时间格式区分

| 来源 | 格式 | 含义 | 示例 |
|------|------|------|------|
| 日记时间块 | MM:SS | 持续时长 | 03:15 = 3分15秒 |
| 文件名 | HH:MM | 时间点 | 11.10 = 11点10分 |
| 录音时间戳 | MM:SS | 相对开始 | 00:03 = 开始后3秒 |

### Git 分支安全操作

**问题**: 从子模块分支创建新分支会带入历史提交，导致 PR 包含过多文件

**解决**: 使用安全流程
1. `git checkout master` - 切换到远程主分支
2. `git pull origin master` - 拉取最新
3. `git checkout -b new-branch` - 创建新分支
4. 恢复/复制工作文件

### 会议时间推算

```
步骤1: 从文件名提取结束时间（如 "11.10 结束" → 11:10）
步骤2: 从录音获取最大时间戳（如 31:30 → 1890秒）
步骤3: 计算开始时间 = 结束时间 - 持续时长
       例: 11:10:00 - 00:31:30 = 10:38:30
```
