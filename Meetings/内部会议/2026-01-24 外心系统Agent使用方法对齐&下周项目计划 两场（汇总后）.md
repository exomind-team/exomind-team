## 一、产品与定位

### 1. 核心定位
- **林嘉濠**：外心系统的核心是把东西变成产品，变成能满足别人用户需要的产品目标
- **林嘉濠**：聚焦某一个功能点率先突破，比如信息聚合、自动处理信息等，先把某个功能点推向产品化
- **吴震宇**：外心系统是"输入→聚合→处理→输出"的闭环
- **林嘉濠**：外心系统聚焦的是生命成长领域，不是Agent编程领域，也不是助手领域

### 2. 差异化价值
- **林嘉濠**：核心竞争力不在技术（技术会被取代），而是以用户为中心的理念，加上与生命相关的哲学思想（如能量感知）
- **林嘉濠**：不是推荐系统，用户可以使用但不会依赖；推荐必须给理由，用户可以记住理由自己执行
- **林嘉濠**：目标是帮助用户从"心理冻土"改造成"净土"或"乐园"

### 3. 竞品分析
- **林嘉濠**：市面上已有很多局域网聊天软件（如Coder Q、Coder）、手机版Mini Agent、可视化Agent工具，这不是卖点
- **林嘉濠**：真正的价值是多个信息源聚合统一处理，以及完整的"人→目标→任务→反馈"流程
- **林嘉濠**：对标产品DI清单、i Done List、AI Todo List等，缺点是需要用户相信它且分析能力有限

### 4. 商业与推广
- **林嘉濠**：Minimix Agent可以分享使用案例获得5000-10000积分（付费宣传换免费额度）
- **林嘉濠**：产品推广需要流量，要学习引流技巧：标题符合流行样式、内容提供安全解决方案、关键词贴合用户痛点
- **林嘉濠**：需要每天输出、不断改进，观察爆款文章并拆解其流量获取方式

## 二、技术架构与实现

### 1. 整体架构
- **林嘉濠**：外心系统数据流设计
  ```
  信息源（QQ/微信等）→ 缓冲区（消息传递引擎）→ Agent处理 → 分类存储 → 输出（Event Log/Time Block/任务）
  ```
- **林嘉濠**：缓冲区现在变成了消息传递引擎，作用是聚合不同来源的信息

### 2. 文件结构（Life OS）
```
human/
├── identity/          # 身份定义
├── goals/            # 目标管理（按心理周期：季/年/周）
├── tasks/            # 任务（按天/周）
├── ideas/            # 想法库（随时记录、回顾）
├── event-logs/       # 事件日志（按物理时间拆分）
├── time-blocks/      # 时间块总结
└── knowledge/        # 知识库（整个知识文件夹）
```

### 3. Event Log设计
- **林嘉濠**：Event Log记录所有原始内容，按物理时间拆分，原始记录永不更改
- **吴震宇**：Event Log要记录：事件ID、时间、内容、实体关系等
- **林嘉濠**：语音识别内容也是派生视图，原始语音是真实记录
- **林嘉濠**：可以删除内容，但删除也要记录事件（或不记录，类比人类遗忘机制）

### 4. 时间块设计
- **林嘉濠**：时间块是25-45分钟执行单元，有开始事件和结束事件
- **林嘉濠**：暂停也是事件（去厕所、洗澡等），所有事件都要如实记录
- **吴震宇**：时间块需要AI总结，总结后删除细节，只保留精炼内容

### 5. 目标与任务
- **林嘉濠**：目标由AI从日志中抽取，人工确认修正
- **林嘉濠**：任务字段：难度（依赖知识链路的复杂程度）、紧急程度、重要性、与目标的关联
- **林嘉濠**：不用评分数字，而是真实表达内部结构，用依赖图/树状图表示

### 6. 技术选型
- **林嘉濠**：技术栈怎么方便怎么来，现在先用TS+Node.js+NPM，最简单最易调试
- **林嘉濠**：参考Terry（AAR的MCP服务器）：IPC电通讯调试、获取前后端接口、截图、点击模拟
- **林嘉濠**：参考Coze AI项目：抽象不同大模型的统一适配层、跨端设计（WebGPU、WGPU等）
- **林嘉濠**：端口要统一规范（2601、2602...），不能让Agent自己生成接口（会冲突）

### 7. 开发流程
- **林嘉濠**：完整的研发流程是团队核心，24-48小时就能从想法变成可用的软件
- **林嘉濠**：GitHub Actions自动化流程：写代码→写文档→写测试→集成测试→图像测试→打包→提交
- **林嘉濠**：现在要直接开发核心流程，不做其他验证了

## 三、Agent设计原则

### 1. Agent定位
- **林嘉濠**：Agent是脚手架，不是替代品；用户可以不用，但用了能更好
- **林嘉濠**：Agent要提供推荐理由，用户能记住理由自己执行
- **林嘉濠**：Agent帮助用户感知自己处于什么状态，避免迷茫

### 2. 记忆系统
- **林嘉濠**：Agent有自己的记忆（短期+长期），不完全依赖外部文件
- **林嘉濠**：Agent的记忆不在产品文档里，是自己整理的踩坑经验
- **林嘉濠**：Agent抽取目标时要确认，用户说不是想要的就要删除并记住这个偏好

### 3. 文件管理
- **林嘉濠**：每个Agent管理自己负责的文件，或有一个统一Agent管理所有文件
- **林嘉濠**：所有内容都是追加写入，修改要生成新事件
- **林嘉濠**：Agent生成的内容都是AI输出，人输入的内容都进Event Log

## 四、外心系统核心流程

### 1. 日循环（每日启动流程）
- **林嘉濠**：每天开始要强制性看今日任务，进行顶层规划
- **林嘉濠**：复习目标（运用复习规律）、心理安慰和鼓励
- **林嘉濠**：习惯系统要定成每日任务，自由时间安排其他事

### 2. 日循环（每日关机流程）
- **林嘉濠**：睡前有关机流程：对接今天的事情、复习、评估效果、分析意识和潜意识内容
- **林嘉濠**：关闭电子系统（手机、电脑），进行轻微冥想、凝神聚气、放松入睡
- **林嘉濠**：关机流程由人自己把控，系统只给轻量级反馈

### 3. 任务执行流程
```
选择任务（今日任务→当下任务）→ 执行任务（记录时间块）→ 反馈（Agent反馈+人工反馈）→ 调节目标/任务
```
- **林嘉濠**：挑任务时，Agent根据当前状态查询身心状态，再推荐任务
- **林嘉濠**：如果状态差，就不建议做任务，建议冥想或休息
- **林嘉濠**：反馈要更新所有前面环节的内容，形成双向循环

### 4. 信息处理流程
- **林嘉濠**：信息入口要便捷（手机语音、电脑剪贴板）
- **林嘉濠**：信息统一放到缓冲区，Agent分类处理，拆分任务
- **林嘉濠**：先打通整个流程，验证流程有没有用，再优化用户体验

## 五、个人身份与目标

### 1. 身份定义
- **林嘉濠**：human文件夹里要写自己的身份、想成为的目标、榜样
- **林嘉濠**：身份定义不是计算机生成的，是人自己写的（90%以上）
- **林嘉濠**：包括：自我定位、资源约束、拥有的工具、能做的事情、想做的事情、当前项目、关注领域、知识储备、能力水平

### 2. 目标管理
- **林嘉濠**：目标分为长期（身份相关）、中期（周/月）、短期（今日任务）
- **林嘉濠**：目标由AI从日志和聊天中总结，人工确认修正
- **林嘉濠**：预期成果要可检验，关联具体行动

### 3. 想法库
- **林嘉濠**：随时记录想法（Idea），定期回顾，Agent也帮忙回顾
- **林嘉濠**：想法命名用内容命名，实现时从想法库抽取
- **林嘉濠**：每个想法都有价值，不会过期

## 六、团队协作

### 1. 角色分工
- **林嘉濠**：林嘉濠更适合探索类工作（写文档、技术选型）
- **吴震宇**：吴震宇更适合优化整合（把需求优化成用户体验最好的方案）
- **林嘉濠**：组合起来上下游打通，吴震宇主负责自己项目，林嘉濠主负责外心系统

### 2. 项目管理
- **林嘉濠**：采用负责人制，一人主导拍板，另一人负责子模块
- **林嘉濠**：可以走不同的技术路线（单Agent或多Agent），但流程必须一致
- **林嘉濠**：先各自尝试，实践后再提出想法，不急于确认

### 3. 会议整理
- **林嘉濠**：整理会议记录时，分类归纳观点，不是原文引用而是转述成更结构化的方式
- **吴震宇**：总结要能回溯，能看到原文出处；去掉语义噪音（重复、赘词）
- **林嘉濠**：可以生成思维导图（树形结构或有向无环图），再转化为文章

## 七、其他观点

### 1. 关于工具
- **林嘉濠**：先做最核心的功能，没有图形界面没关系，能用就行
- **林嘉濠**：参考Cloud的思路，核心是能做事情，不是界面多漂亮
- **林嘉濠**：插件管理功能已完成，可以展示和分享

### 2. 关于隐私
- **林嘉濠**：Event Log里可以有隐私内容，用户可以自己删除
- **林嘉濠**：删除是用户自己的决定，系统不干预
- **吴震宇**：可以保留"曾有这个内容"的记录但隐藏具体内容，作为保底机制

### 3. 关于写作
- **林嘉濠**：写作要贴近用户，不是自言自语；向毛主席、艾思奇学习文风，让泥腿子也能看懂
- **林嘉濠**：每天写日记锻炼写作能力，半小时即可
- **林嘉濠**：宣传渠道：V2EX、简书、知乎、集智空间等

### 4. 关于休息
- **林嘉濠**：家务等生活事务是Agent很难代劳的，除非有机器人
- **林嘉濠**：Agent睡觉时也能帮忙做一些事情（代替一部分劳动力）
- **林嘉濠**：外心系统先聚焦能打通的领域，小车项目（硬件）成本高，以后再说
