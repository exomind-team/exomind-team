## 🔧 方法论提炼

### 第一层：基础整理（去噪）

#### 1.1 单行格式转换

**来源**：集体库 `01-录音转写整理-单行格式提示词.md`

**核心规则**：
- 每条发言一行：`发言人，时间戳，内容`
- 去除所有空行
- 合并同一发言人的多行内容
- 时间戳保持原文格式

**发言人规范化**：
```
星林 → 林嘉濠
wzy → 吴震宇
```

#### 1.2 口语去噪

**来源**：transcript-cleaner-skill 迭代报告

**规则分类**：

| 规则类型 | 规则示例 | 处理效果 |
|----------|----------|----------|
| **单字重复** | `有有` → `有`，`我我我` → `我` | 去除口语冗余 |
| **连续重复词** | `来到了这个点来到这个点` → `来到了这个点` | 合并重复片段 |
| **冗余表达** | 删除 `或者说`，`传统或者说先前的` → `传统先前的` | 简化表达 |
| **特殊模式** | `他他它` → `他`，`约等于睡眠约` → `约等于睡眠` | 修正错误重复 |

**处理统计（本次整理）**：
- 原始行数：1318 行
- 整理后行数：~420 行
- 压缩比例：~68%

### 第二层：结构化（摘要）

#### 2.1 会议摘要

**来源**：集体库 `03-会议摘要-科学家式分析提示词.md`

**输出模板**：

```markdown
> [!summary] 会议摘要、总结与科学家式分析（YYYY-MM-DD HH:mm）
> **参会**: ...
> **录音起点**: ...
> **核心主题**: ...
>
> **摘要**
>
> - ...
>
> **总结（综合）**
>
> - ...
>
> **科学家式分析**
>
> - **研究问题**: ...
> - **关键假设**: ...
> - **证据线索**: ...
> - **机制解释**: ...
> - **可检验预测**: ...
> - **实验/验证建议**: ...
> - **风险与限制**: ...
>
> **待办清单（原文引用）**
>
> - [ ] "..."
```

#### 2.2 关键原则

| 原则 | 说明 |
|------|------|
| **原文引用** | 待办必须来自原文，用引号标注 |
| **不补写** | 不添加原文未出现的信息 |
| **中英对照** | 首次出现的英文术语加中文备注 |
| **区分来源** | 区分人的输入和 AI 的反馈 |

### 第三层：洞察（分析）

#### 3.1 观点分类

**方法**：从整理后的单行格式中提取各方观点

**分类维度**：

| 维度 | 说明 |
|------|------|
| **共识点** | 双方都同意的内容 |
| **分歧点** | 双方观点不一致的地方 |
| **待定项** | 需要后续验证的内容 |
| **行动项** | 明确的待办任务 |

#### 3.2 话题树

**方法**：识别谈话的话题展开结构

│   └── AI 虚拟形象

    ├── CI/CD 自动化
