## 1. 核心观点提取（来自1.10-1.12日记）

### 1.1 L1-L3三层架构

| 层级 | 名称 | 环境 | 核心特征 |
|-----|------|------|---------|
| L1 | 基因/细胞层 | 内存空间 | 最小认知生命体，争夺CPU时间片 |
| L2 | 组织/个体层 | 操作系统 | 多个L1组成的群体，处理宏观数据 |
| L3 | 具身层 | 物理世界 | 传感器连续数据降维映射到L1/L2 |

### 1.2 关键概念定义

- **能量** = CPU执行步数(Step) + IDLE包(空闲时间)
- **身体** = 占据的内存空间 + 读写范围
- **边界** = 读/写范围限制（类似细胞膜）
- **死亡** = 步数耗尽、边界被侵蚀
- **食物** = IDLE包，吃掉后转化为步数

### 1.3 核心机制

1. **食物链机制**：生命体需争夺IDLE包维持存在
2. **攻击/防御**：读写操作可攻击其他生命体边界
3. **分裂繁殖**：能量充足时可分裂成两个
4. **死亡机制**：步数上限、空间膨胀失控、被侵蚀

## 2. 与会人员反馈摘要

### 2.1 刘凯导师反馈

**认可点**：
- 表征的多维混合设计（数据意义随读取方式变化）
- 自下而上的建构原则

**质疑点**：
- 奥卡姆剃刀：是否需要这么多底层的"小生命"？
- 能量维持：每个节点即使空闲也消耗计算资源
- 机器人vs生物：无法实现分布式能量系统

### 2.2 吴震宇建议

- 引入死亡机制防止死循环（癌细胞问题）
- 步数上限约束迫使生命体"运动"
- 环境激励设计（生存游戏/解谜）

### 2.3 亚亚困惑

- 身体定义：数据和操作系统如何作为"身体"？
- 具身性：如何体现具身认知？

## 3. 推演案例设计决策

### 3.1 选择听觉通道原因

| 考虑因素 | 视觉 | 听觉 | 选择 |
|---------|------|------|-----|
| 数据复杂度 | 高(像素矩阵) | 低(频谱) | 听觉 |
| 实时性 | 30fps | 44KHz | 听觉 |
| 会议讨论 | 未明确 | 明确提及音色 | 听觉 |
| 实现难度 | 高 | 中 | 听觉 |

### 3.2 推演流程设计

```
L3物理层 → L2 OS层 → L1细胞层 → 行动 → 反馈
  麦克风    听觉Agent  生命体群    电机    能量闭环
```
