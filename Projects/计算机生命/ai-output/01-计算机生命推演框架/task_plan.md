## 3. 核心推演内容

### 3.1 L1-L3架构定义（来自日记）

```
L1 基因/细胞层
├── 身体：占据的内存空间(RAM)
├── 能量：CPU执行步数(Step) + IDLE包
└── 机制：类似Tierra/Avida，争夺CPU时间片

L2 组织/个体层
├── 环境：操作系统(OS)
└── 功能：L1演化出的群体，处理宏观数据

L3 具身层
├── 环境：物理世界(AGI Car)
└── 挑战：传感器数据降维映射到L1/L2
```

### 3.2 推演案例：听觉音色识别

```
场景：二轮小车识别"充电桩提示音"

流程：
L3: 麦克风采集 → ADC → 内存映射
    ↓
L2: 听觉Agent拆分频段 → 分发给各L1生命体
    ↓
L1: 每个生命体处理一个频段 → 识别音色模式
    ↓
L2: 聚合各L1结果 → 决策
    ↓
L3: 电机控制 → 小车行动
    ↓
能量闭环：消耗步数 → 觅食IDLE包补充
```

### 3.3 失败场景

| 场景 | 触发条件 | 结果 |
|------|---------|------|
| 步数耗尽 | 步数→0 | 死亡，躯壳遗留 |
| 边界侵蚀 | 被其他生命体写穿 | 死亡 |
| 空间崩溃 | 体积膨胀失控 | 被吃掉 |
