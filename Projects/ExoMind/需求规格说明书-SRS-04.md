## 1. 交互设计指南

### 1.1 核心设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **默认短** | 输出和操作保持极简 | 复盘只4行；开机<10分钟 |
| **默认可跳过** | 所有仪式/建议可跳过 | 关机仪式可跳过 |
| **失败不羞辱** | 记录为"未完成"而非"失败" | 中断计时不惩罚 |
| **最小干预** | 界面极简，不打断流程 | 状态栏常驻，点击展开 |
| **渐进式呈现** | 先关键判断，后细节展开 | Scout先输出三问 |

### 1.2 操作时间约束

| 操作 | 最大时间 |
|------|----------|
| 开始计时 | ≤10秒 |
| 查看状态 | ≤3秒 |
| 复盘输入 | ≤2分钟 |
| 开机仪式 | ≤10分钟 |
| 关机仪式 | ≤10分钟 |

### 1.3 交互模式定义

| 模式 | 触发方式 | 响应方式 |
|------|----------|----------|
| **即时反馈** | 用户操作 | <100ms响应 |
| **渐进呈现** | 内容加载 | 先骨架后细节 |
| **可恢复中断** | 任意时刻 | 保存上下文 |

## 3. 多端适配原则

| 端 | 适配策略 |
|----|----------|
| **PC (Tauri)** | 完整功能 + 窗口管理 + 快捷键 |
| **Android** | 简化界面 + 蓝牙连接 + 系统分享入口 |
| **Web** | 完整功能 + PWA支持 |
| **CLI/TUI** | 文本界面 + 键盘导航 |

## 4. 界面隐喻

| 概念 | 隐喻 | 说明 |
|------|------|------|
| ExoMind | 外骨骼/新器官 | 辅助而非替代人类 |
| Governor | 方向盘+刹车 | 系统调度与治理 |
| Scout | 前线侦察兵 | 信息侦察与预警 |
| Time Block | 时间容器 | 可回放行动闭环 |
| EventLog | 事实记录仪 | 不可篡改的历史 |
