# 系统架构与数据流

## 目标

- 描述"多入口、单事实"的结构
- 说明主干流程与关键对象

## 主干流程

```
外部缓冲区 → 身心状态觉察 → 任务管理 → 执行 → 反馈
   (1)           (2)          (3)      (4)    (5)
```

当前阶段：**外部缓冲区**（Scout 侦察兵）

## 核心架构原则

1. **Truth Layer（事实层）**：EventLog = 日记（按时间顺序记录自己的事实，不可修改）
2. **Surface Layer（入口界面层）**：多入口（Obsidian、浏览器、手机等）统一到单一真相

原则：Surface 可以很多；Truth 只能有一个。

## 关键对象

- **EventLog（日记）**：发生过什么的可审计历史。日记是其实践。
- **Session / TimeBlock**：可启动的行动闭环（主干流程第4阶段）
- **ExoBuffer（外部缓冲区）**：Scout 侦察兵前哨站（主干流程第1阶段）
