# 核心闭环与场景

## 目标

- 明确外心的最小闭环协议
- 给出最小可执行场景

## 核心闭环协议

1. 目标：我想去哪里
2. 觉察：我现在在哪里（含身心与情境）
3. 行动：以 Session/TimeBlock 组织行动 【人类hailay：存疑，未理解，需要讨论】
4. 反馈/校验：发生了什么、是否有效
5. 巩固：沉淀为知识点/脚本/习惯触发器

**关键声明**：休息也很重要，在工作时间块间隙自动生成并给予休息引导。【人类hailay：关键点，需要讨论】

## 最小可执行场景

- 外部缓冲区：统一收集输入，减少入口碎片化
- Session / TimeBlock：把行动组织成可回放的闭环
- Now View：对齐方向、目标与当下行动
- 复盘与反馈：以 EventLog 为依据进行复盘与校验
