# ExoMind MVP 需求

> **核心原则**：只记录 + 计时，其他后续再说。

---

## 1. MVP 功能范围

| 功能       | MVP范围            | 后续扩展          |
| -------- | ---------------- | ------------- |
| **消息记录** | 输入框 → 追加EventLog | 搜索、过滤、AI总结    |
| **时间块**  | 计时 → EventLog    | 正/倒计时、统计、能量评分 |
| **开机仪式** | 可选，跳过不羞辱         | 完整仪式流程        |
| **关机仪式** | 可选，跳过不羞辱         | 完整仪式流程        |

---

## 2. 用户故事 (MVP)

### US-01：记录事件

| 要素  | 内容             |
| --- | -------------- |
| 场景  | 发生事件/想到什么/收到信息 |
| 输入  | 文本（可选语音转文字）    |
| 输出  | 追加到EventLog    |
| 原则  | 只追加、不修改、不删除    |

### US-02：开始计时

| 要素  | 内容           |
| --- | ------------ |
| 场景  | 开始一个任务/学习/工作 |
| 输入  | 任务名称（可选）     |
| 输出  | 计时开始、状态栏显示   |
| 原则  | 操作 ≤10秒      |

### US-03：停止计时

| 要素  | 内容                       |
| --- | ------------------------ |
| 场景  | 完成/中断任务                  |
| 输入  | 完成/中断（可选原因）              |
| 输出  | EventLog(time-block.end) |
| 原则  | 允许中断、不羞辱                 |

---

## 3. 数据结构 (MVP)

### 3.1 EventLog 格式

```yaml
---
type: message | time-block
id: evt-YYYYMMDD-NNN
ts: YYYY-MM-DD HH:mm:ss
content: "消息内容/任务名称"
---
```

### 3.2 实体关系

```
EventLog（唯一事实层）
  ├─ type: message | time-block
  ├─ id: 唯一标识
  ├─ ts: 时间戳
  └─ content: 内容
```

### 3.3 存储位置

| 端       | 存储        | 路径                  |
| ------- | --------- | ------------------- |
| Android | SQLite    | /data/data/exomind/ |
| PC      | SQLite    | ~/.exomind/         |
| Web     | IndexedDB | 浏览器本地               |

## 5. 技术约束

| 约束   | 要求     |
| ---- | ------ |
| 启动时间 | ≤3秒    |
| 操作响应 | ≤100ms |
| 离线可用 | 必须     |
| 数据本地 | 必须     |

---

## 6. MVP 验收标准

- [ ] 可以输入文字并保存为EventLog
- [ ] 可以开始/停止计时
- [ ] 计时数据保存为EventLog
- [ ] 数据本地持久化
- [ ] 离线可用

---

*MVP目标：最小可运行版本，验证核心闭环*
