# 工程需求分析

**自我反馈**不应该评分，只留给用户一个输入框即可。以后根据用户输入的对自我的评价的文本，我们调用 agent 进行自动评价，或者拟合近似评分。

**时间块的消息**可以**不启动时间块就记录消息**，然后，两个时间块启动间隔，中间那些消息就会被视为一个自动生成的时间块而被保留。因为中间的间隔可以视为一个休息或者是工作别的事情去了，而且可以后续再划分时间块也是可以的。但是不是现在做 MVP，先不做这些功能，只是它允许能够不启动时间块就开始发送消息，这是肯定是要允许的。

发消息的目的，就是只要人发生了事件，或想记录信息，一切都要**原封不动**地保留下来。
这是**事实层**必须做的事。
